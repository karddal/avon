# System architecture

Avon was designed to be modular and easily extensible. 

## Main components

There are two main parts of the system.

[Frontend](frontend.md)
  : The frontend of Avon, written in Next.js and Typescript.

[Backend](backend.md)
  : The backend server, written in Python, using FastAPI and SQLModel.

The frontend and the backend communicate using a REST API. Automatically-generated OpenAPI-compliant API docs are automatically generated by the system, when running locally, they can be reached at `localhost:8000/docs`.

More information about how this communication works and how it is organised can be found on the main [frontend](frontend.md) page.

## External components

Avon relies on several external components to be able to provide its services.

[GitLab](gitlab.md)
  : Either a cloud or self-hosted instance of GitLab is used to provide repository hosting.

[AWS](aws.md)
  : AWS is used both to deploy our project, and for the test running system.
